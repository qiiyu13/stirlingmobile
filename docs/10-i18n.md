```
app/src/main/res/
├── values/
│   └── strings.xml            ← Default (English), source of truth
├── values-ar/
│   └── strings.xml
├── values-de/
│   └── strings.xml
├── values-es/
│   └── strings.xml
├── values-fr/
│   └── strings.xml
├── ... (one per language)
└── values-zh-rCN/
    └── strings.xml
```

### String Naming Convention

```
tool_{id}_{element}_{context}

Examples:
  tool_merge_title = Merge PDFs
  tool_merge_description = Combine multiple PDF files into a single document
  tool_merge_input_label = Select PDF files to merge
  tool_merge_processing = Merging files...
  tool_merge_success = Merged successfully
  tool_merge_error_invalid = One or more selected files are not valid PDFs

  category_page_operations = Page Operations
  category_convert = Convert
  category_security = Security

  error_generic = Something went wrong
  error_password_required = This PDF requires a password
  error_password_wrong = Incorrect password
  error_file_too_large = File is too large (max {size}MB)
  error_unsupported_format = Unsupported file format
```

### Pluralization

```xml
<!-- Android quantity strings -->
<plurals name="tool_merge_file_count">
    <item quantity="one">1 file selected</item>
    <item quantity="other">%d files selected</item>
</plurals>

<plurals name="pdf_page_count">
    <item quantity="one">1 page</item>
    <item quantity="other">%d pages</item>
</plurals>
```

## 3. Translation Process

### 3.1 Source of Truth

`values/strings.xml` (English) is the canonical source. All other translations derive from it.

### 3.2 Tooling

```bash
# Extract all English strings as JSON for translators
./gradlew extractTranslations

# Import translated JSON back to XML
./gradlew importTranslations --lang de --file translations_de.json

# Check for missing translations
./gradlew checkTranslations  # CI step — fails if any key is missing in any locale
```

### 3.3 Translation Format

Translators receive JSON:

```json
{
  "tool_merge_title": {
    "en": "Merge PDFs",
    "de": "PDFs zusammenführen"
  },
  "tool_merge_input_label": {
    "en": "Select PDF files to merge",
    "de": "PDF-Dateien zum Zusammenführen auswählen"
  }
}
```

### 3.4 Contribution

Translations via Crowdin / GitHub PRs. Each PR must:
- Only modify ONE language
- Not modify `values/strings.xml` (English)
- Include all new keys since last translation update

## 4. RTL Layout

### 4.1 Compose Support

Compose handles RTL automatically via `LayoutDirection`. All screens use:

```kotlin
@Composable
fun AppTheme(content: @Composable () -> Unit) {
    MaterialTheme(
        content = content,
        layoutDirection = if (isRtlLanguage()) LayoutDirection.Rtl else LayoutDirection.Ltr
    )
}

fun isRtlLanguage(): Boolean {
    val lang = Locale.getDefault().language
    return lang in setOf("ar", "fa", "he", "ur", "ps", "sd", "ku", "yi")
}
```

### 4.2 RTL Considerations

- Tool icons that imply direction (rotate right/left, reorder arrows) must flip in RTL
- Page navigation arrows must reverse in RTL
- File list ordering (left-to-right grid) must reverse
- PDF viewer pages: left-to-right reading vs right-to-left reading (respect PDF metadata)

## 5. Locale-Aware Formatting

| Data | Formatter | Example |
|---|---|---|
| File size | `java.text.NumberFormat` | "2.4 MB" (en) / "2,4 MB" (de) |
| Page numbers | `java.text.NumberFormat` | "Page 1,234" (en) / "Page 1.234" (de) |
| Dates | `java.time.format.DateTimeFormatter` with locale | "Jan 15, 2026" (en) / "15. Jan. 2026" (de) |
| Percentages | `java.text.NumberFormat.getPercentInstance()` | "75%" (en) / "75 %" (fr) |

## 6. OCR Languages

OCR language packs are separate from UI translations. They are Tesseract `traineddata` files:

```
/ocr/tessdata/
├── eng.traineddata     (English, bundled in APK)
├── deu.traineddata     (German, download on demand)
├── fra.traineddata     (French)
├── jpn.traineddata     (Japanese)
├── ara.traineddata     (Arabic)
├── ... (40+ languages)
```

Downloaded from: `https://github.com/tesseract-ocr/tessdata/raw/main/{lang}.traineddata`

Language pack downloads happen over the network. This is the **only** network access in the app, and only when the user explicitly requests it. Per NF-020 (00-spec.md), this download is isolated in its own dynamic feature module carrying the sole INTERNET permission — the base module and all other tools remain network-free. OCR can work offline with already-downloaded packs.
